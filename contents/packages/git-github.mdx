### First-Time Config

<CodeBlock language="md">
{`# Set your name for commit attribution
git config --global user.name "Your Name"

# Set your email for commit attribution (should match your GitHub email for verified commits)
git config --global user.email "your@email.com"


# Set VS Code as the default editor for Git (e.g., for commit messages, rebase, etc.)
git config --global core.editor "code --wait"

# üî• Useful Git Aliases

# Show pretty commit graph (great for visualizing branching)
git config --global alias.graph "log --all --decorate --oneline --graph"

# Short status output with branch info and staged/unstaged files
git config --global alias.st "status -sb"
`}
</CodeBlock>

### üì¶ Repository Basics

Starting a Project

<CodeBlock language="md">

### üîÑ Daily Commands

<CodeBlock language="md">
{`# Check status
git status

# Stage changes
git add file.txt
git add .              # All files
git add -p             # Interactive staging

# Commit changes
git commit -m "Message"
git commit -am "Msg"   # Add & commit tracked files
git commit --amend     # Fix last commit

# View history
git log
git log -p             # With changes
git log --since="2 weeks ago"
git log --author="Name"`}
</CodeBlock>

### üåø Branching Strategies
Branch Operations

<CodeBlock language="md">
{`# Create branch
git branch feature-x
git checkout -b hotfix  # Create & switch

# List branches
git branch              # Local
git branch -r           # Remote
git branch -a           # All

# Delete branches
git branch -d branch    # Local
git push origin --delete branch  # Remote`}
</CodeBlock>

### ü§ù Collaboration Workflows

**Fork & PR Flow**

1. Fork repository on GitHub

2. Clone your fork locally

3. Add upstream remote:

<CodeBlock language="md">
{`git remote add upstream https://github.com/original/repo.git`}
</CodeBlock>

4. Sync with upstream:

<CodeBlock language="md">
{`git fetch upstream
git merge upstream/main`}
</CodeBlock>

### Team Collaboration

<CodeBlock language="md">
{`# Fetch changes
git fetch origin

# Merge strategies
git merge branch        # Regular merge

git rebase branch      # Linear history

git merge --squash branch  # Single commit

# Resolve conflicts
git mergetool          # Use configured tool

git add resolved-file

git commit`}
</CodeBlock>
